# 実装基準メモ

参照モック: `docs/mockups/mobile-ui-mockup.png`

## 画面方針

- スマホ縦長画面を主対象にする。
- PCではスマホUIを中央寄せで見せ、横に広げすぎない。
- トップ、診断、結果、図鑑の各画面は、1画面目で完成度が伝わるようにする。
- 背景は暗い古代ジャングル、DNA、琥珀、化石、夜のネオン感を組み合わせる。
- 可愛いマスコット恐竜と、迫力あるリアル寄り恐竜の両方を使う前提でレイアウトする。
- 実画像未配置時は、CSS/Canvasのプレースホルダーで破綻させない。

## UI方針

- 操作要素は親指で押しやすい高さにする。
- 診断ページは1問ずつ表示し、5択カードを縦に並べる。
- 進捗は数値とバーの両方で見せる。
- 角丸、細い発光ライン、琥珀/エメラルド/ターコイズ/赤のアクセントを使う。
- 仮実装感のある単純なカードUIではなく、スマホゲーム風の密度と立体感を出す。

## 結果カード方針

- 画面上の結果カードも、保存画像も9:16を基準にする。
- 保存画像は `1080x1920px` のPNG。
- 表示要素は、サイト名、診断日、タイプ名、恐竜名、キャッチコピー、恐竜ビジュアル、5軸スコア、レーダーチャート、強み、注意ポイント。
- ダウンロード画像はスマホ壁紙/SNS投稿に流用しやすい余白と文字サイズにする。

## 実装上の注意

- GitHub Pagesで動く相対パスにする。
- 外部API通信は行わない。
- 回答データは外部送信しない。
- MBTI、16Personalities等の名称やデザインには寄せない。
- スマホ幅 360px / 390px / 430px で横スクロールを出さない。
