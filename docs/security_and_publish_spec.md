# 公開・安全設計

## 1. GitHub Pages公開
- 静的サイトとして公開する。
- 相対パスを使う。
- public配下またはassets配下に画像を配置する。
- 大容量画像はWebP化して軽量化する。

## 2. 権利・表現上の注意
- MBTIの名称をサイト名や正式診断名として使わない。
- MBTI公式設問、16Personalities風の説明文、他社デザインを流用しない。
- 「独自ロジックによるエンタメ性格診断」と明記する。
- 恐竜の事実情報を書く場合は、断定しすぎず一般的な説明に留める。

## 3. プライバシー
- 個人情報は入力させない。
- 回答データは外部送信しない。
- localStorageを使う場合は端末内保存のみ。

## 4. セキュリティ
- 外部入力をHTMLとして直接挿入しない。
- JSONデータは信頼済みのローカルファイルのみ使用する。
- 画像保存処理で外部URLを読み込まない。
- npmパッケージを使う場合は最小限にする。

## 5. 将来拡張時の注意
- SNS共有を実装する場合、共有URLやOGP生成の方針を別途設計する。
- アクセス解析を入れる場合、プライバシーポリシーを用意する。
