# 恐竜タイプ診断サイト 全体仕様

## 1. サイト概要
ユーザーが複数の質問に回答し、5つの性格軸のスコアから16種類の恐竜タイプに分類される診断サイト。

## 2. ターゲット体験
- 診断として「当たっている」と感じられること。
- 結果画面を保存したくなること。
- 可愛さとカッコよさが両立していること。
- スマホで気持ちよく遊べること。

## 3. 公開方式
- GitHub Pagesで公開する。
- バックエンドなし。
- HTML / CSS / JavaScriptのみ、またはVite構成でも可。
- 外部API通信は不要。
- 結果画像保存はブラウザ内で完結させる。

## 4. ページ構成
### 4.1 トップページ
- サイト名
- キャッチコピー
- メインビジュアル
- 診断開始ボタン
- 診断の説明
- 注意書き: 「この診断は独自ロジックによるエンタメ性格診断です」

### 4.2 診断ページ
- 1問ずつ表示
- 進捗バー
- 選択肢は5択を基本とする
- 戻るボタン
- 回答完了後に結果ページへ遷移

### 4.3 結果ページ
- 恐竜タイプ名
- キャッチコピー
- リアル寄り恐竜イラスト
- マスコット恐竜イラスト
- 5軸レーダーチャート
- 性格説明
- 強み
- 注意ポイント
- 相性の良いタイプ
- 近いタイプ
- 結果画像保存ボタン
- もう一度診断ボタン

### 4.4 図鑑ページ
- 16タイプ一覧
- 各タイプの概要
- 結果詳細へのリンク

## 5. デザイン方針
- スマホファースト。
- 恐竜・古代・冒険感を軸にする。
- ただし古臭くせず、現代的なポップUIにする。
- 色味は「深いネイビー/黒系 + 琥珀/エメラルド/ターコイズ/赤」のアクセント。
- 丸みのあるUIで可愛さ、背景やライティングでカッコよさを出す。

## 6. 非機能要件
- スマホで横スクロールが出ない。
- 画像はWeb向けに圧縮する。
- GitHub Pagesでパス崩れしないよう相対パスを使う。
- 画像が未配置でも壊れないようプレースホルダー表示を用意する。
- localStorageは途中回答保存程度に限定する。
