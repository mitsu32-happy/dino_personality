# Codexへの統合指示

あなたは、GitHub Pagesで公開する「恐竜タイプ診断サイト」を実装してください。

## 最重要方針
- docs/配下の設計書を正本として扱う。
- 不明点があっても、既存設計から安全に補完できる範囲は自律的に実装する。
- MBTI、16Personalities等の名称・設問・デザイン・説明文は使用しない。
- 独自の5軸スコアリング診断として作る。
- 可愛いとカッコイイを両立する。
- GitHub Pagesで動作する静的サイトにする。
- スマホファーストで作る。

## 実装前に行うこと
1. docs/ui_mock_prompt.md をもとに、image2.0でサイト全体モックを作成する前提で、実装デザインの基準を定義する。
2. モック画像がある場合は、それを最優先のUI基準としてHTML/CSSに落とし込む。
3. モック画像がない場合は、docs/site_spec.md と docs/asset_spec.md の方針に従って高品質UIを実装する。

## 必須機能
- トップページ
- 診断ページ
- 結果ページ
- 図鑑ページ
- 24問診断
- 5軸スコア計算
- 16タイプ判定
- レーダーチャート表示
- 結果画像保存ボタン
- もう一度診断
- 画像未配置時のプレースホルダー表示

## データ
- data/questions.json を読み込んで設問を表示する。
- data/dino_types.json を読み込んで結果タイプを表示する。
- data/scoring_rules.json を参考に判定する。

## 結果判定
1. 回答ごとに5軸へ加点する。
2. 合計点を算出する。
3. 上位2軸を求める。
4. dino_types.jsonのprimaryAxis/secondaryAxisと照合する。
5. 完全一致がなければ、primaryAxis一致を優先し、secondaryAxisが近いタイプを選ぶ。
6. 同点時は直近10問の軸スコア、次にpriorityで決める。

## UI/UX
- 診断中は1問ずつ表示する。
- 進捗バーを表示する。
- 回答カードは押しやすくする。
- 結果画面は保存したくなる見た目にする。
- 画面遷移は軽いアニメーションを入れる。
- スマホで横スクロールを出さない。

## アセット
- 以下のパスを想定する。
  - assets/images/dinos/real/{type_id}.webp
  - assets/images/dinos/mascot/{type_id}.webp
  - assets/images/cards/bg_{type_id}.webp
  - assets/images/ui/logo.svg
- 画像がない場合はCSSグラデーションやプレースホルダーで破綻しないようにする。

## 結果カード保存
- HTML上の結果カードを画像化して保存できるようにする。
- html2canvas等の利用は可。ただしGitHub Pagesで動く構成にする。
- 依存を増やしすぎない。

## 品質基準
- Lighthouseで大きな問題が出ない程度に軽量化する。
- デザインは仮実装感を出さない。
- 文言は自然な日本語にする。
- 診断として納得感が出るよう、判定理由とスコアを見せる。

## 完了条件
- ローカルでトップ→診断→結果→画像保存まで確認できる。
- GitHub Pages公開時にパス崩れしない。
- 16タイプ図鑑が表示される。
- 画像未配置でも表示が崩れない。
- READMEに起動方法・公開方法・アセット追加方法を書く。
